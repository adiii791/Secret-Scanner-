What is Docker?
- Platform for building, shipping, running applications
- Packages app + dependencies in container
- Runs same everywhere

## Key Terms
- **Image**: Blueprint (like recipe)
- **Container**: Running instance (like cooked meal)
- **Dockerfile**: Instructions to build image
- **Docker Hub**: Store for images (like GitHub for containers)

## Why Docker?
- Works on any machine
- Consistent environment
- Easy deployment
- Industry standard

## Commands I'll use:
- docker build → Create image
- docker run → Start container
- docker ps → See running containers
- docker images → See all images
